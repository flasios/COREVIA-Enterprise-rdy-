Section 01

Architecture Principles

Five non-negotiable invariants that govern every decision in the COREVIA Brain.

Governance-First

No AI reasoning happens before policy clearance at Layer 3. Every request must pass through policy enforcement before touching any engine.

Append-Only Decision Object

The DecisionObject is never mutated. Each layer appends its section. Full audit trail from intake to memory.

No Side-Effects Before L7

Only Layer 7 and above can persist to external systems. Layers 1–6 are read-analyze-only.

No Learning From Rejected

Engine C only ingests APPROVED decisions. Rejected or draft outputs never enter the distillation pipeline.

Classification Drives Everything

Data classification at L2 determines engine routing, redaction mode, HITL requirements, allowed regions, and retention period.

Sovereign Data Never Leaves

SOVEREIGN-classified data can never route to an external engine. Enforced at L2 constraints, Engine Router, and IPLAN validation.

L1

Intake

Normalize raw input into canonical IntakeData
Deep-trim strings, strip nulls, capture serviceId, routeKey, userId, timestamp
L2

Classification

Detect data sensitivity and set security constraints
Keyword scan → Sovereign / Confidential / Internal / Public. Sets: allowCloudProcessing, allowExternalModels, requireHitl, allowedRegions, maxRetentionDays. Detects sector (government, healthcare, finance, defense, education, infrastructure).
L3

Policy Ops

Policy-as-code enforcement gate
4 built-in policies (classification access, budget threshold, sovereign protection, service-specific). Registry packs from DB. Authority matrix: CISO+CTO+Minister → auto-approved based on risk × budget.
Can BLOCK pipeline
L4

Context & Completeness

Score input quality and gather enterprise context
Section-weighted scoring: Vision 30%, Resources 15%, Technology 15%, Required 40%. Quality gate ≥ 0.4. Ambiguity detection for "maybe", "TBD", "approximately". Generation routes bypass quality gate.
Can request NEEDS_INFO
L5

Orchestration

Build the Intelligence Plan (IPLAN) — the contract for L6
40+ use-case keys → blueprints. Engine Router resolves primary/fallback engines. Plugin Registry selects models. Output Schema Registry binds response format. Agent authorization checked. IPLAN packaged with token/time/cost budgets.
L6

Reasoning — SAFE ZONE

Execute governed intelligence tasks
IPLAN LAW enforced: no Engine C, redaction required for external. Execution order: Engine A → Engine B → Agents. Progressive enhancement: agent baseline + LLM enrichment. Draft artifacts created with versioning. 60s SLA / 120s timeout.
L7

Validation & HITL

Quality gate before output delivery
5 threshold checks: confidence ≥ 60, minimum 2 options, risk data present, evidence present, budget ≤ 10M AED. 3 bias detectors: single option, extreme score gap, high recommendation despite risk. Generates APR-{ts}-{nanoid} approval IDs.
Can pause for HITL review
L8

Memory & Learning

Persist decision and trigger post-approval actions
Creates decision summary, gathers evidence, extracts rationale, generates tags. Post-approval: portfolio_sync agent executes. Engine C distillation scheduled (runs separately, not in pipeline).

Engine A — Sovereign Internal

Local only · Zero external network calls
  • Ollama runtime (Mistral / DeepSeek / Llama)
  • RAG search via pgvector
  • Rule-based scoring & prioritization
  • Entity extraction from input
  • Historical pattern matching
  • Local LLM artifact generation
  • Policy-aware reasoning
  • Secure document analysis

Engine B — Hybrid Augmented

External LLMs via redaction gateway
  • Claude Sonnet (primary provider)
  • GPT-4o (fallback provider)
  • Structured options generation
  • Risk analysis & assessment
  • Strategic assessment
  • Executive content writing
  • Dual-provider consensus (high-value)
  • Circuit breaker per plugin

Engine C — Distillation

Post-approval only · Never runs in L6
  • Pattern extraction from decisions
  • Best practice harvesting
  • Template improvement
  • Scoring weight updates
  • Routing insight refinement
  • Policy test case generation
  • Training sample preparation
  • All outputs start as DRAFT

Section 07

Agent Development Kit (ADK)

Registered agents that execute within L6 under IPLAN authorization. Each agent is authorized per-request by L5.

Demand Agent

Rule-based field synthesis. Infers department, project name, stakeholders, challenges, outcomes. Zero LLM calls, <1ms.

Quality Gate Agent

Validates output completeness and quality. Checks all required fields populated, scores confidence.

Pack Builder Agent

Assembles capability and knowledge packs for downstream consumption.

Risk Controls Agent

Risk assessment and control identification based on classification, sector, and budget.

Evidence Collector

Gathers decision support evidence from RAG results, engine outputs, and prior decisions.

Portfolio Sync Agent

Post-approval agent (L8). Syncs approved demand into portfolio management system. Mode: execute.

Section 08

Safety Infrastructure

Three safety gateways that protect data sovereignty and system resilience.
GatewayPurposeApplies ToMechanism
Redaction Gateway Mask PII/sensitive data before external LLM calls Engine B (all external calls) Pattern-based entity detection. Reversible masking with token mapping. Sovereign: aggressive mode. Confidential: standard mode.
RAG Gateway Semantic search across enterprise knowledge Engine A (knowledge retrieval) pgvector embeddings via local-embeddings service. Query → embed → cosine similarity → ranked results.
Circuit Breaker Protect against cascading failure from external APIs Engine B (per-plugin tracking) Failure threshold → open circuit → cooldown → half-open → test → close. Per-plugin state. Auto-failover to alternate provider.
